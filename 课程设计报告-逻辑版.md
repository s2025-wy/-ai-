# 基于AI的大学生职业规划智能体 - 课程设计报告

---

## 1、系统概述

### 1.1 项目背景
随着高等教育的普及和就业市场竞争的加剧，大学生面临着日益严峻的就业压力。如何科学地进行职业规划，选择适合自己的职业方向，成为大学生亟待解决的问题。本项目旨在利用人工智能技术，为大学生提供一个集职业规划、岗位探索、简历分析、智能咨询于一体的综合性平台。

### 1.2 项目简介
本系统是一个基于AI的大学生职业规划智能体平台，采用前后端分离架构，集成了大语言模型、知识图谱等先进技术。系统通过分析学生的个人信息、专业背景、兴趣爱好等，为其提供个性化的职业规划建议、岗位推荐、面试准备等服务，帮助大学生更好地进行职业发展规划。

### 1.3 技术架构
- **前端技术栈**：Vue 3 + TypeScript + Element Plus + Pinia + Vue Router + ECharts
- **后端技术栈**：Python 3.10+ + FastAPI + SQLAlchemy + SQLite + Neo4j
- **AI技术**：LangChain + DashScope API（大语言模型）
- **移动端**：uni-app（支持微信小程序、App等多端）

### 1.4 系统特点
1. **AI驱动**：集成大语言模型提供智能职业咨询和个性化建议
2. **知识图谱**：使用Neo4j构建岗位关系网络，可视化展示职业发展路径
3. **个性化推荐**：基于学生画像提供定制化的职业规划方案
4. **全栈架构**：前后端分离，技术栈现代化，易于扩展和维护
5. **多端支持**：同时支持Web端和移动端，满足不同场景使用需求

## 2、功能模块分析及数据结构描述

### 2.1 用户模块

#### 2.1.1 功能描述
用户模块是整个系统的基础模块，负责用户的注册、登录、个人信息管理、权限验证等功能。

#### 2.1.2 核心功能
- **用户注册**：填写用户名、邮箱、密码等信息注册新账户
- **用户登录**：支持用户名密码登录，集成验证码验证
- **个人信息管理**：编辑个人基本信息、上传头像
- **密码修改**：修改登录密码
- **权限管理**：区分普通用户和管理员权限

#### 2.1.3 数据结构
用户模块的数据结构包含用户的基本信息、认证信息和权限信息。主要包括用户ID、用户名、邮箱、姓名、性别、年龄、电话、专业、年级、角色、头像、密码哈希值和个人资料完成度等字段。这些信息存储在用户数据库表中，用于用户身份验证和个人信息管理。

### 2.2 首页模块

#### 2.2.1 功能描述
首页模块是用户进入系统后的第一界面，提供系统功能导航、最新岗位推荐、快捷入口等功能。

#### 2.2.2 核心功能
- **功能导航**：提供各主要功能模块的快捷入口
- **岗位推荐**：基于用户画像智能推荐合适的岗位
- **系统公告**：展示最新系统通知和公告
- **快捷操作**：提供常用功能的快速访问

### 2.3 商品管理模块（岗位管理模块）

#### 2.3.1 功能描述
岗位管理模块是系统的核心功能模块之一，负责岗位信息的展示、搜索、筛选、申请、收藏等功能。

#### 2.3.2 核心功能
- **岗位列表**：展示所有岗位信息，支持分页显示
- **岗位搜索**：支持按关键词、行业、地点等条件搜索岗位
- **岗位详情**：展示岗位的详细信息，包括薪资、要求、职责等
- **岗位申请**：申请感兴趣的岗位，系统记录申请状态
- **岗位收藏**：收藏喜欢的岗位，方便后续查看
- **岗位画像**：展示岗位的技能要求、职业发展路径
- **知识图谱**：可视化展示岗位之间的关联关系

#### 2.3.3 数据结构
岗位模块的数据结构包含岗位的基本信息、要求和描述。主要包括岗位ID、岗位名称、公司名称、薪资、工作地点、工作经验、学历要求、岗位描述、技能要求、标签、行业、岗位类型和发布日期等字段。此外，还包含用户申请记录和收藏记录的数据结构，用于跟踪用户与岗位的互动。

### 2.4 用户管理模块

#### 2.4.1 功能描述
用户管理模块是管理员专用模块，负责对系统用户进行管理，包括用户列表查看、用户信息编辑、用户删除等功能。

#### 2.4.2 核心功能
- **用户列表**：展示所有注册用户信息，支持分页和搜索
- **用户详情**：查看用户的详细信息
- **用户编辑**：编辑用户信息，修改用户角色、重置密码等
- **用户删除**：删除用户账户，清理相关记录
- **岗位管理**：添加、编辑、删除岗位信息

### 2.5 学生画像模块

#### 2.5.1 功能描述
学生画像模块负责收集和管理学生的个人信息、专业背景、能力评估等，为职业规划提供数据基础。

#### 2.5.2 核心功能
- **个人信息管理**：编辑姓名、专业、年级等基本信息
- **简历上传**：上传PDF、Word格式的简历文件，系统自动分析内容
- **能力分析**：基于简历和测评结果分析能力，生成能力雷达图
- **职业目标设定**：设定个人的职业发展目标

#### 2.5.3 数据结构
学生画像模块的数据结构包含学生的基本信息、专业背景和职业目标。主要包括学生ID、用户ID、姓名、专业、年级、职业目标和当前规划进度等字段。这些信息与用户信息关联，形成完整的学生画像，为个性化推荐和职业规划提供数据支持。

## 3、主要算法或流程描述

### 3.1 用户注册与登录流程

#### 3.1.1 用户注册流程

**流程说明：**
1. 用户访问注册页面，填写用户名、邮箱、密码等个人信息
2. 系统验证输入信息的合法性，包括格式、长度等要求
3. 系统检查用户名和邮箱是否已被其他用户注册
4. 对用户输入的密码进行加密处理，确保密码安全
5. 将用户信息保存到数据库中
6. 返回注册成功信息，引导用户登录

**流程图：**
```
开始
  ↓
用户填写注册信息
  ↓
验证输入信息格式
  ↓
检查用户名是否已存在
  ↓ 是
返回"用户名已被注册"
  ↓ 否
检查邮箱是否已存在
  ↓ 是
返回"邮箱已被注册"
  ↓ 否
加密用户密码
  ↓
保存用户信息到数据库
  ↓
返回注册成功
  ↓
结束
```

#### 3.1.2 用户登录流程

**流程说明：**
1. 用户访问登录页面，系统自动生成验证码图片
2. 用户填写用户名、密码和验证码
3. 系统验证验证码是否正确且未过期
4. 系统验证用户名是否存在于数据库中
5. 系统验证密码是否与数据库中存储的加密密码匹配
6. 验证通过后，生成JWT访问令牌
7. 返回令牌和用户信息，完成登录

**流程图：**
```
开始
  ↓
生成验证码图片
  ↓
用户输入登录信息（用户名、密码、验证码）
  ↓
验证验证码是否存在且未过期
  ↓ 否
返回"验证码已过期"
  ↓ 是
验证验证码是否正确
  ↓ 否
返回"验证码错误"
  ↓ 是
查询用户是否存在
  ↓ 否
返回"用户不存在"
  ↓ 是
验证密码是否正确
  ↓ 否
返回"密码错误"
  ↓ 是
生成JWT访问令牌
  ↓
返回令牌和用户信息
  ↓
结束
```

### 3.2 岗位管理流程

#### 3.2.1 岗位搜索与浏览流程

**流程说明：**
1. 用户进入岗位探索页面
2. 系统加载所有岗位列表数据
3. 用户可以输入关键词、选择行业或地点等筛选条件
4. 系统根据筛选条件构建数据库查询
5. 执行查询并返回符合条件的岗位列表
6. 用户点击岗位卡片查看详细信息

```

  ↓

  ↓
加载所有岗位列表
  ↓
用户是否设置筛选条件？
  ↓ 是
构建查询条件（关键词、行业、地点）
  ↓
执行数据库查询
  ↓
返回筛选后的岗位列表
  ↓
用户点击岗位
  ↓
查询岗位详细信息
  ↓
展示岗位详情页面
  ↓
结束
```

#### 3.2.2 岗位申请与收藏流程

**岗位申请流程：**
1. 用户查看岗位详情页面
2. 点击"申请岗位"按钮
3. 系统验证用户是否已登录
4. 检查用户是否已申请过该岗位
5. 创建申请记录，设置状态为"待处理"
6. 返回申请成功信息

**岗位收藏流程：**
1. 用户查看岗位详情或在岗位列表中
2. 点击"收藏"按钮
3. 系统验证用户是否已登录
4. 检查用户是否已收藏过该岗位
5. 创建收藏记录
6. 返回收藏成功信息

### 3.3 用户管理流程

## 4、系统使用说明

### 4.1 环境要求

#### 4.1.1 后端环境
- Python 3.10 或更高版本
- pip 包管理工具
- SQLite数据库（无需额外安装）
- Neo4j图数据库（可选，用于知识图谱功能）

#### 4.1.2 前端环境
- Node.js 20.19.0 或更高版本
- npm 或 yarn 包管理工具

#### 4.1.3 移动端环境
- HBuilderX 开发工具
- 微信开发者工具（用于微信小程序调试）

### 4.2 系统部署

#### 4.2.1 后端部署

1. **进入后端目录**
2. **安装依赖**：使用pip安装requirements.txt中的依赖包
3. **配置环境变量**：创建.env文件，配置密钥、API密钥等
4. **初始化数据库**：运行数据库迁移脚本
5. **创建管理员账户**：运行管理员初始化脚本
6. **启动服务**：运行main.py或使用start.bat批处理文件

后端服务将在http://localhost:8000启动，API文档可访问http://localhost:8000/docs。

#### 4.2.2 前端部署

1. **进入前端目录**
2. **安装依赖**：使用npm install安装依赖包
3. **配置API地址**：修改配置文件中的API地址为后端服务地址
4. **开发模式运行**：使用npm run dev启动开发服务器
5. **生产环境构建**：使用npm run build构建生产版本

构建完成后，将dist目录部署到Web服务器即可。

#### 4.2.3 移动端部署

1. **使用HBuilderX打开项目**：打开app端目录
2. **配置网络地址**：修改配置文件中的API地址
3. **运行到微信小程序**：使用HBuilderX的运行功能，选择微信开发者工具
4. **打包发布**：使用HBuilderX的发行功能，按照提示完成打包和发布流程

## 5、问题及解决办法

### 5.1 开发过程中遇到的问题

#### 问题1：前后端跨域问题

**问题描述：**
前端运行在http://localhost:5173，后端运行在http://localhost:8000，由于浏览器的同源策略，前端无法正常访问后端API。

**解决办法：**
在后端添加CORS中间件，允许跨域请求。通过配置CORS中间件，设置允许的源、方法和头部，使得前端能够正常访问后端API。

#### 问题2：数据库迁移问题

**问题描述：**
在修改数据模型后，需要重新创建数据库表，导致原有数据丢失。

**解决办法：**
使用Alembic进行数据库版本管理，实现增量迁移。通过创建迁移脚本，记录数据库结构的变化，避免每次修改模型都重新创建表结构。

#### 问题3：大模型API调用失败

**问题描述：**
调用DashScope API时经常出现超时或网络错误。

**解决办法：**
1. 添加重试机制，当API调用失败时自动重试
2. 设置合理的超时时间，避免长时间等待
3. 使用异步请求提高并发处理能力，减少阻塞

#### 问题4：图片验证码显示问题

**问题描述：**
在不同操作系统上，验证码图片的字体渲染不一致，部分文字无法正常显示。

**解决办法：**
实现字体fallback机制，优先使用系统字体，失败时使用默认字体。通过尝试加载不同的字体，确保在不同系统上都能正常显示验证码。

### 5.2 系统运行常见问题

#### 问题1：后端启动失败，提示端口被占用

**解决办法：**
1. 检查8000端口是否被其他程序占用
2. 修改配置文件中的端口号
3. 或关闭占用端口的程序

#### 问题2：前端页面空白或加载失败

**解决办法：**
1. 检查后端服务是否正常启动
2. 确认API地址配置正确
3. 清除浏览器缓存
4. 检查浏览器控制台的错误信息

#### 问题3：文件上传失败

**解决办法：**
1. 检查文件大小是否超出限制
2. 确认文件格式是否支持
3. 检查uploads目录的写入权限
4. 查看后端日志获取详细错误信息

#### 问题4：登录时验证码错误

**解决办法：**
1. 确认验证码输入正确（不区分大小写）
2. 检查验证码是否过期（5分钟有效期）
3. 点击刷新按钮获取新的验证码
4. 确保浏览器支持Cookie
