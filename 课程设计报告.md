# 基于AI的大学生职业规划智能体 - 课程设计报告

---

## 1、系统概述

### 1.1 项目背景
随着高等教育的普及和就业市场竞争的加剧，大学生面临着日益严峻的就业压力。如何科学地进行职业规划，选择适合自己的职业方向，成为大学生亟待解决的问题。本项目旨在利用人工智能技术，为大学生提供一个集职业规划、岗位探索、简历分析、智能咨询于一体的综合性平台。

### 1.2 项目简介
本系统是一个基于AI的大学生职业规划智能体平台，采用前后端分离架构，集成了大语言模型、知识图谱等先进技术。系统通过分析学生的个人信息、专业背景、兴趣爱好等，为其提供个性化的职业规划建议、岗位推荐、面试准备等服务，帮助大学生更好地进行职业发展规划。

### 1.3 技术架构
- **前端技术栈**：Vue 3 + TypeScript + Element Plus + Pinia + Vue Router + ECharts
- **后端技术栈**：Python 3.10+ + FastAPI + SQLAlchemy + SQLite + Neo4j
- **AI技术**：LangChain + DashScope API（大语言模型）
- **移动端**：uni-app（支持微信小程序、App等多端）

### 1.4 系统特点
1. **AI驱动**：集成大语言模型提供智能职业咨询和个性化建议
2. **知识图谱**：使用Neo4j构建岗位关系网络，可视化展示职业发展路径
3. **个性化推荐**：基于学生画像提供定制化的职业规划方案
4. **全栈架构**：前后端分离，技术栈现代化，易于扩展和维护
5. **多端支持**：同时支持Web端和移动端，满足不同场景使用需求

## 2、功能模块分析及数据结构描述

### 2.1 用户模块

#### 2.1.1 功能描述
用户模块是整个系统的基础模块，负责用户的注册、登录、个人信息管理、权限验证等功能。

#### 2.1.2 核心功能
- 用户注册：支持用户名、邮箱、密码等信息注册
- 用户登录：支持用户名密码登录，集成验证码验证
- 个人信息管理：用户可编辑个人基本信息、上传头像等
- 密码修改：支持用户修改登录密码
- 权限管理：区分普通用户和管理员权限

#### 2.1.3 数据结构

## 3、主要算法或流程描述

### 3.1 用户注册与登录流程

#### 3.1.1 用户注册流程

**流程说明：**
1. 用户访问注册页面，填写用户名、邮箱、密码等信息
2. 系统验证输入信息的合法性（格式、长度等）
3. 检查用户名和邮箱是否已被注册
4. 对密码进行加密处理（使用bcrypt算法）
5. 将用户信息保存到数据库
6. 返回注册成功信息

**流程图：**
```
开始
  ↓
用户填写注册信息
  ↓
验证输入信息格式
  ↓
检查用户名是否已存在
  ↓ 是
返回"用户名已被注册"
  ↓ 否
检查邮箱是否已存在
  ↓ 是
返回"邮箱已被注册"
  ↓ 否
加密用户密码
  ↓
保存用户信息到数据库
  ↓
返回注册成功
  ↓
结束
```

#### 3.1.2 用户登录流程

**流程说明：**
1. 用户访问登录页面，系统生成验证码图片
2. 用户填写用户名、密码和验证码
3. 系统验证验证码是否正确且未过期
4. 验证用户名是否存在
5. 验证密码是否正确
6. 生成JWT访问令牌
7. 返回令牌和用户信息

**流程图：**
```
开始
  ↓
生成验证码图片
  ↓
用户输入登录信息（用户名、密码、验证码）
  ↓
验证验证码是否存在且未过期
  ↓ 否
返回"验证码已过期"
  ↓ 是
验证验证码是否正确
  ↓ 否
返回"验证码错误"
  ↓ 是
查询用户是否存在
  ↓ 否
返回"用户不存在"
  ↓ 是
验证密码是否正确
  ↓ 否
返回"密码错误"
  ↓ 是
生成JWT访问令牌
  ↓
返回令牌和用户信息
  ↓
结束
```

1. **进入后端目录**
```bash
cd 后端
```

2. **安装依赖**
```bash

```

3. **配置环境变量**
创建 `.env` 文件，配置以下内容：
```

DASHSCOPE_API_KEY=your-dashscope-api-key
NEO4J_URI=bolt://localhost:7687
```

## 5、问题及解决办法

### 5.1 开发过程中遇到的问题

#### 问题1：前后端跨域问题

**问题描述：**
前端运行在 `http://localhost:5173`，后端运行在 `http://localhost:8000`，由于浏览器的同源策略，前端无法正常访问后端API。

**解决办法：**
在后端添加CORS中间件，允许跨域请求：
```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

#### 问题2：数据库迁移问题

**问题描述：**
在修改数据模型后，需要重新创建数据库表，导致原有数据丢失。

**解决办法：**
使用Alembic进行数据库版本管理，实现增量迁移：
1. 初始化Alembic
2. 创建迁移脚本
3. 执行迁移升级

#### 问题3：大模型API调用失败

**问题描述：**
调用DashScope API时经常出现超时或网络错误。

**解决办法：**
1. 添加重试机制
2. 设置合理的超时时间
3. 使用异步请求提高并发处理能力

#### 问题4：图片验证码显示问题

**问题描述：**
在不同操作系统上，验证码图片的字体渲染不一致，部分文字无法正常显示。

**解决办法：**
实现字体 fallback 机制，优先使用系统字体，失败时使用默认字体：
```python
try:
    font = ImageFont.truetype("arial.ttf", 28)
except:
```

### 5.2 系统运行常见问题

#### 问题1：后端启动失败，提示端口被占用

**解决办法：**
1. 检查8000端口是否被其他程序占用
2. 修改 `main.py` 中的端口号
3. 或关闭占用端口的程序

#### 问题2：前端页面空白或加载失败

**解决办法：**
1. 检查后端服务是否正常启动
2. 确认API地址配置正确
3. 清除浏览器缓存
4. 检查浏览器控制台的错误信息

#### 问题3：文件上传失败

**解决办法：**
1. 检查文件大小是否超出限制
2. 确认文件格式是否支持
3. 检查 `uploads` 目录的写入权限
4. 查看后端日志获取详细错误信息

#### 问题4：登录时验证码错误

**解决办法：**
1. 确认验证码输入正确（不区分大小写）
2. 检查验证码是否过期（5分钟有效期）
3. 点击刷新按钮获取新的验证码
4. 确保浏览器支持Cookie
